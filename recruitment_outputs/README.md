# Recruitment report outputs

This directory contains dated aggregate reports generated by
`recruitment_reports.py` for NANO and NICO. The canonical run writes:

- one standalone HTML milestone report per study;
- one combined Excel workbook with milestone, live-as-of, data-quality,
  provenance, date-candidate, and QA sheets; and
- one JSON run manifest containing aggregate counts and validation results.

Participant identifiers and row-level classifications must never be written
here. They belong only in the ignored `recruitment_audit_secure/` directory.

Public artifacts automatically show participant-derived cells from 1 through
9 as `<10`. Data-quality and date-coverage counts are published only in
ten-person bands, and temporally complementary cumulative cells receive
secondary suppression, to reduce subtraction risk. Exact current diagnostics
remain local in the secure audit workflow.

Date-field coverage in these reports is diagnostic. A populated visit, birth,
evaluation, or optional-consent date is not treated as a protocol-enrollment
date unless the study owner explicitly verifies that meaning.

Aggregate output is not automatically approved for public dissemination.
Apply the study's IRB and data-use rules before sharing.
